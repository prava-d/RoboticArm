The Powerful Pi

Benjamin Ziemann :: November 3, 2017

With the addition of wanting to use computer vision to differentiate objects and predict their materials, we knew we would need more than the Arduino microprocessor. However we still wanted the entire project to be self contained so we went with integrating a Raspberry Pi.

We went with the Raspberry Pi as it had a few key features:

  • We had (limited) experience with it
  • It had a large community around it
  • OpenCV and TensorFlow have been proven to work on it
  • Web capabilities for interfacing with a web app

While the setup was pretty standard, the one big deviation we had to make was setting a static ip. Because we wanted to consistently connect with the Pi after turning it on and off, we didn’t have to have a DHCP server IP. Instead with a static one we could reliably use the same IP to connect via SSH for setup and file transfer and later VNC for debugging.

While initially brainstorming for more unique ways to take a robotic arm project, before settling on grip strength variability, several of our ideas involved tele-operational control. As part of integration of a Raspberry Pi we thought the addition of a web app to control the gripper would be a good idea, both for manual testing purposes as well as possible future features.

To do this we took advantage of the static IP and had the Pi host a python based Flask web app on it’s own ip. By taking in certain HTTP requests, we could return specific pages or have the Pi run on board tasks like sending commands via serial to the Arduino.

Because all it takes is an HTTP request, phones could also act like a remote control if you knew the proper route.
